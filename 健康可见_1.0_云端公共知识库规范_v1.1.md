# 《健康可见 1.0 云端公共知识库规范 v1.1》

## 1. 适用范围
1. 适用于本项目云端“公共知识库”建设、更新、检索、审计。
2. 不包含用户个人健康数据存储；个人数据仅用于运行时检索，不反写公共库。
3. 输出仅为证据与候选建议，不替代医生诊断与处方。

## 2. 三层知识库定义
1. `L1 临床知识库（最高信度）`：公开可获取且公认医学来源（指南、共识、权威数据库等）。
2. `L2 溯因知识库（中信度）`：生物学、生理学、微生物学、免疫学、病理学等机制知识。
3. `L3 生活方式/功能医学知识库（探索层）`：生活方式干预、功能医学相关知识。
4. 三层均含两类条目：`判断类`、`手段类`。

## 3. 统一字段字典（最小必填）
1. `kb_id`：知识条目唯一ID。
2. `layer`：L1/L2/L3。
3. `claim_type`：judgment/procedure。
4. `topic`：主题。
5. `statement`：核心陈述（单条原子化）。
6. `applicability`：适用条件。
7. `contraindication`：禁忌/不适用。
8. `evidence_level`：证据等级（A/B/C 或 GRADE）。
9. `confidence_score`：0-1。
10. `source_name`：来源名称。
11. `source_url`：来源链接。
12. `source_version`：来源版本号/版次。
13. `publication_date`：发布日。
14. `ingest_date`：入库日。
15. `review_status`：draft/reviewing/approved/rejected/deprecated。
16. `reviewer`：审核人。
17. `supersedes_id`：被替代条目ID。
18. `valid_from`：生效时间。
19. `valid_to`：失效时间。
20. `tags`：标签。
21. `citation_span`：证据定位片段。
22. `checksum`：内容哈希。
23. `audit_trail_id`：审计链路ID。

## 4. 原子化与向量化策略
1. 必做原子化：每条只表达一个可检索事实或手段。
2. 必做结构化索引：topic、claim_type、evidence_level、contraindication 必可过滤。
3. 建议向量化：按条目原子做 embedding，不按整篇文档做单向量。
4. 检索采用混合模式：结构化过滤 + 关键词 + 向量召回 + 重排。
5. 分层控权：L1 结果优先，L2 补机制，L3 仅补充并显式标注。

## 5. 入库审核清单（打勾制）
1. 来源是否在白名单。
2. 来源是否可公开复核。
3. 是否存在商业推广/转载污染风险。
4. 条目是否原子化且无多结论混写。
5. 是否填写适用条件与禁忌。
6. 证据等级是否标注。
7. citation_span 是否可定位。
8. 与现有同主题条目是否冲突。
9. 冲突是否完成人工裁决。
10. 审核状态是否为 approved。
11. 审计记录是否完整。
12. 未通过任一项即禁止上线检索。

## 6. 污染防控机制
1. 白名单源头管理：机构/期刊/数据库按批次审批。
2. 黑名单机制：匿名内容、无出处二次转载、营销软文禁入。
3. 自动去重：checksum + 语义近重复检测。
4. 冲突检测：同主题相反结论自动告警。
5. 人工抽检：每周新增抽检、每月高频命中抽检。

## 7. 更新与版本制度
1. 不覆盖更新：仅新增版本，旧版本转 deprecated。
2. 发布节奏建议：L1 周更月发版，L2 双周更，L3 月更。
3. 撤稿/失效触发：自动降级并通知复核。
4. 支持快照回滚：按发布日期恢复上个稳定版。

## 8. 检索打分公式（建议）
1. 总分：`S = 0.40*R_struct + 0.25*R_keyword + 0.25*R_vector + 0.10*R_quality`
2. 分层权重因子：`W_layer(L1)=1.00, W_layer(L2)=0.80, W_layer(L3)=0.60`
3. 证据权重因子：`W_evidence(A)=1.00, W_evidence(B)=0.85, W_evidence(C)=0.70`
4. 最终分：`S_final = S * W_layer * W_evidence`
5. 冲突惩罚：若与高层级冲突，`S_final = S_final * 0.5`。
6. 展示规则：默认 TopN 中至少 1 条 L1；无 L1 时必须显示“仅中低信度证据”。

## 9. 前端展示强制项
1. 必显示层级标签（L1/L2/L3）。
2. 必显示来源、版本、日期、证据等级。
3. 手段类条目必显示适用条件与禁忌。
4. 冲突条目必显示“存在冲突，需人工确认”。

## 10. 上线验收标准
1. 100% 在线条目可追溯到 source_url 与版本。
2. 100% 在线条目含 review_status=approved。
3. 100% 展示含层级与证据信息。
4. 抽检污染率与误引率达项目阈值。
5. 检索日志与引用日志可按 audit_trail_id 全链路回放。
