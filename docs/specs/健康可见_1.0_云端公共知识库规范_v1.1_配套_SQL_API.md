# 《健康可见 1.0 云端公共知识库规范 v1.1》配套：SQL 表结构 + API 契约（入库 / 检索 / 审计）

## 0. 适用与实现约定
- 数据库：PostgreSQL 15+
- 向量：pgvector 扩展
- 时间：统一存储为 `timestamptz`（UTC），前端按本地时区展示
- 多租户：每条业务数据均包含 `tenant_id`，用于“本项目云端专用知识库”隔离
- 响应外层统一结构：`status / message / data / error_hint`

## 1. SQL 表结构（PostgreSQL）

### 1.1 扩展与枚举
```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS pg_trgm;

DO $$
BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_type WHERE typname = 'kb_layer') THEN
    CREATE TYPE kb_layer AS ENUM ('L1', 'L2', 'L3');
  END IF;

  IF NOT EXISTS (SELECT 1 FROM pg_type WHERE typname = 'kb_claim_type') THEN
    CREATE TYPE kb_claim_type AS ENUM ('judgment', 'procedure');
  END IF;

  IF NOT EXISTS (SELECT 1 FROM pg_type WHERE typname = 'kb_review_status') THEN
    CREATE TYPE kb_review_status AS ENUM ('draft', 'reviewing', 'approved', 'rejected', 'deprecated');
  END IF;

  IF NOT EXISTS (SELECT 1 FROM pg_type WHERE typname = 'kb_result_type') THEN
    CREATE TYPE kb_result_type AS ENUM ('success', 'partial', 'failed', 'manual_needed');
  END IF;

  IF NOT EXISTS (SELECT 1 FROM pg_type WHERE typname = 'kb_job_status') THEN
    CREATE TYPE kb_job_status AS ENUM ('queued', 'running', 'succeeded', 'failed', 'canceled');
  END IF;
END$$;
```

  ON kb_entry_versions USING GIN (tsv);

  ON kb_entry_versions (tenant_id, review_status, valid_from, valid_to)

## 2. API 契约

## 2.1 通用约定
- Base URL：`/kb/v1`
- 认证：`Authorization: Bearer <token>`
- 必填 Header：
  - `X-Tenant-Id: <uuid>`
  - `X-Request-Id: <uuid>`
- 统一响应外层：
```json
{
  "status": "success|partial|failed|manual_needed",
  "message": "human readable message",
  "data": {},
  "error_hint": "optional"
}
```

## 3. 检索评分落地规则

## 3.1 公式
- `S = 0.40*R_struct + 0.25*R_keyword + 0.25*R_vector + 0.10*R_quality`
- `S_final = S * W_layer * W_evidence * P_conflict`

其中：
- `W_layer(L1)=1.00, W_layer(L2)=0.80, W_layer(L3)=0.60`
- `W_evidence(A)=1.00, W_evidence(B)=0.85, W_evidence(C)=0.70`
- `P_conflict=0.5`（与高层级冲突时），否则为 `1.0`

## 3.2 展示约束
- TopN 默认至少 1 条 L1；无 L1 时必须返回 `error_hint`：`only_mid_low_confidence_evidence`。
- L3 条目必须带标签：`exploratory_non_firstline`。

## 4. 错误码契约（建议）

| code | 场景 | 说明 |
|---|---|---|
| KB-4001 | 参数错误 | 缺必填字段/枚举非法 |
| KB-4002 | 租户隔离错误 | tenant 不匹配 |
| KB-4003 | 来源未白名单 | source blocked |
| KB-4004 | 审核状态不允许 | 非 approved 不能上线检索 |
| KB-4041 | 资源不存在 | source/document/entry/version 不存在 |
| KB-4091 | 冲突未解决 | high-severity conflict open |
| KB-4221 | 污染风险过高 | pollution_risk_score 超阈值 |
| KB-5001 | 检索引擎异常 | 混合检索失败 |
| KB-5002 | 向量服务异常 | embedding 生成失败 |

## 5. 最小上线清单（SQL + API）
1. SQL：完成 `kb_tenants/kb_sources/kb_documents/kb_entries/kb_entry_versions/kb_audit_logs/kb_query_logs/kb_hit_logs`。
2. API：至少上线
   - 入库：`POST /sources`, `POST /documents`, `POST /entries`, `POST /reviews/{version_id}/approve`
   - 检索：`POST /retrieve/search`
   - 审计：`GET /audit/logs`
3. 运行要求：所有线上可检索条目必须 `review_status=approved` 且可追溯 `source_url + source_version + citation_span`。
